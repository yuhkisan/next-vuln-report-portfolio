# AGENTS

## 作業フロー（必須）

- 作業を始める前に、必ず GitHub Issue を立てる。
- PR と Issue の粒度は揃える（1PR = 1Issue を基本）。
- PR 作成時に本文へクローズキーワード（例: `Closes #123`）を必ず記載し、default ブランチへのマージで自動クローズさせる。
- PR がマージされずクローズされた場合、Issue はクローズしない（必要なら理由を Issue にコメント）。
- 複数 Issue が関係する場合は、PR 本文に全てのクローズキーワードを記載する。
- PR 化できない作業（GitHub 設定変更など）は、事前に Issue で合意を取り、実施後に結果/理由を Issue に記録する。
