# 設計思想（簡易）

- SSR/CSR はページ単位で分ける（静的/SEOはSSR、管理/操作中心はCSR）。
- SSR→CSR の初期値注入は原則避け、必要な場合のみ例外として扱う。
- SSR は Prisma 直取得を基本とし、同一リクエスト内の軽い重複は許容する。
- 速度問題が出たら最適化する（N+1回避、集約、必要ならキャッシュ）。
- CSR は API + データフェッチライブラリで統一する。
- 取得する側がエラーハンドリングまで責任を持つ。
- props は必要な最小データだけ渡す。
- 型注釈は境界でのみ明示し、内部は推論を優先する。
- union は可読性のため明示する。
- 抽象化は慎重に行い、誤った抽象化より重複を許容する。
- コロケーションは機能単位で完結させる。
